# ADR-001: 단방향 레이어 아키텍처

**상태**: 확정
**날짜**: 2026-03-27
**결정자**: 박성훈

## 맥락

HAchilles는 다양한 출력 형식(터미널, JSON, HTML)을 지원하고 진단 로직의 단위 테스트가 가능해야 한다. 또한 HAchilles 자신이 하네스 엔지니어링의 메타 사례여야 하므로 레이어 아키텍처를 강제로 적용한다.

## 결정

의존성 방향을 단방향으로 고정한다:

```
models ← scanner ← auditors ← score ← prescriptions ← report ← cli
```

각 레이어는 자신보다 하위 레이어(왼쪽)만 import할 수 있다.

## 근거

1. **테스트 용이성**: Auditor를 ScanResult 픽스처로 독립 테스트 가능
2. **CLI 독립성**: 터미널/JSON/HTML 출력 형식이 진단 로직에 영향을 미치지 않음
3. **메타 일관성**: HAchilles 자체가 AC-05 (의존성 방향 위반 0건) 기준을 통과해야 함
4. **확장성**: 새 출력 형식(예: Slack 알림)을 cli 레이어에만 추가하면 됨

## 결과

- 각 모듈 docstring 첫 줄에 "레이어 규칙: ..." 명시 의무화
- AGENTS.md에 위반 금지 패턴으로 등록
- CI에서 `import-linter` 또는 동등한 도구로 검증 예정 (Sprint 3)

## 대안 검토

- **DI 컨테이너 패턴**: 유연하지만 복잡도 증가. 현재 규모에서 불필요.
- **플러그인 아키텍처**: Phase 2 사용자 정의 Auditor에서 재검토 예정.
